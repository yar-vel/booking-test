# Booking Service (Test Task)

Микросервис для высоконагруженного бронирования билетов.

## Запуск

```bash
docker-compose up -d
```

## Решение Double Booking

Проблема решена на уровне базы данных.
Создана отдельная таблица `reservations`, в которой `seat_id` является первичным ключом.
Вставить в таблицу несколько резерваций для одного места просто не получится.

## Нагрузочное тестирование

Для имитации нагрузки в 50 000 человек добавлен скрипт на базе `autocannon`.
Он генерирует 50к запросов с разными `user_id` на одно и то же место.

```bash
npm run test:load
```
